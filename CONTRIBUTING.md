# Contributing to AssetFlow

First off, thank you for considering contributing to AssetFlow! 

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues to see if it has already been reported. When creating a bug report, use the provided `Bug report` template.

### Suggesting Enhancements

We are always looking for ways to make AssetFlow better. Use the `Feature request` template to suggest enhancements.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`npm run test`).
5. Make sure your code lints (`npm run lint`).
6. Issue that pull request!

## Local Development

Please see the [README.md](README.md) for instructions on setting up the project locally. We use Husky, Prettier, and ESLint to maintain code quality.
